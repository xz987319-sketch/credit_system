### 2. 解决表单数据残留问题
- **问题**：用户退出表单后再进入，旧数据仍然显示
- **根因**：Session 只在完全无数据时清空，无法区分"首次进入"和"从后续步骤返回"
- **解决方案**：引入 `max_step_reached` session 变量
  - 首次进入 step=0 且 max_step_reached=0 → 清空 session
  - 之后进入 step=0 且 max_step_reached>0 → 保留数据
  - 每次点击"下一步"时更新 max_step_reached

### 5. 修复多步骤表单必填校验失效问题
- **问题**：银行专用栏（最后一页）必填字段未校验，点击提交直接弹出确认框
- **根因1**：`multi_step_form.html` 中 select/radio/checkbox/file 字段的 `required` 属性渲染条件不完整
  - text 等字段：`{% if field.is_required and not field.is_readonly %}required{% endif %}` ✅
  - select/radio/checkbox/file：`{% if field.is_required %}required{% endif %}` ❌（缺少 `not field.is_readonly`）
- **修复1**：统一所有字段类型的 `required` 渲染逻辑，添加 readonly 检查和禁用样式
- **涉及文件**：`apply/templates/multi_step_form.html`
  - select 字段（第302-314行）
  - radio 字段（第316-331行）
  - checkbox 字段（第333-346行）
  - file 字段（第348-367行）
- **根因2**：`post_confirm.js` 调用 `validateFormOnSubmit` 时未使用 `await`
  - `validateFormOnSubmit` 是 `async` 函数，返回 Promise
  - `!Promise` 永远为 `false`，导致校验永远被跳过
- **修复2**：将 submit 事件处理函数改为 `async`，使用 `await` 调用校验函数
- **涉及文件**：`static/js/post_confirm.js`
