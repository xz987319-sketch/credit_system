## 拼音API问题排查与修复（21:00-22:25）

### 问题1：PC端拼音API调用404
- 原因：PC端JS使用 `/employee/api/pinyin/`，实际URL是 `/api/pinyin/`
- 修复：multi_step_form.html 中改为 `/api/pinyin/`

### 问题2：H5姓名拼音失焦无反应
- 原因：H5模板中 `field.field_type == 'text'` 在 `field.validation_rule == 'name'` 之前判断，导致姓名字段被当作普通text处理
- 修复：h5_multi_step_form.html 中将姓名条件移到text条件之前（第173行）

### 问题3：H5拼音字段样式问题
- 修改：去掉输入框样式，改为纯文本div显示，添加 `overflow-x:auto` 支持滚动

### 问题4：添加姓名≥2个汉字验证
- PC端：validateName函数改为检查 `chineseCount < 2`
- H5端：updatePinyinField函数中添加 `chineseCount < 2` 判断

### 问题5：PC/H5页面禁用浏览器返回功能
- 问题：申请人提交后用返回按钮回到上一页重新提交，导致数据错乱
- 修复：在multi_step_form.html和h5_multi_step_form.html开头添加：
  ```javascript
  history.pushState(null, '', location.href);
  window.addEventListener('popstate', function(event) {
      history.pushState(null, '', location.href);
  });
  ```
