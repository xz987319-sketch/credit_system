### 今日修改

2. **自动填充字段红框清除**：当身份证号解析后自动填充出生日期和性别时，**主动调用 clearInvalid() 清除红框**

### 补充资料页面改动（return_edit.html）

1. **姓名校验函数** `validateName` 完整规则：·后≥2汉字、至少2汉字、不超过15汉字、只能输入汉字
2. **`showFieldError` / `hideFieldError` 修复**：JS创建的错误div加 `data-jk-error` 标记，`hideFieldError` 只清除JS创建的，不清除后端返回的错误。**根因**：后端校验失败重渲染后，input事件触发hideFieldError会错误清除后端错误提示
3. **后端加姓名格式校验**：`employee.py` 的 `_validate_page_for_return_edit` 加了姓名格式校验分支（field_type==name or validation_rule==name）
4. **后端联系人手机校验**：条件改为 `field_type == 'phone' or field.validation_rule == 'phone'`，覆盖直系/其他联系人手机
5. **错误div显示问题修复**：
   - `_dynamic_fields.html`：姓名字段加 `js-name-field` class；错误div加 `style="display: block;" data-backend-error="1"` 确保可见
   - `return_edit.html`：加CSS `.card-body .text-danger.small.mt-1[data-backend-error] { display: block !important; }` 覆盖Bootstrap隐藏规则，精准定位，不影响其他页面

### H5表单页历史守卫增强（h5_multi_step_form.html）
- SUBMITTED_KEY：区分是否已提交
- FORM_INVALID_KEY：区分是否已拦截过一次
- 第一次拦截（从成功页返回）：显示5秒倒计时覆盖层，结束后设置 FORM_INVALID_KEY，跳转银行大厅
- 第二次及之后拦截：显示"表单已失效"页面（独立HTML，不走表单页逻辑）
